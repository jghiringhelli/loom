# experiments/proofs — Proof Experiments Index

Each subfolder is a self-contained Loom program that proves one theoretical claim.
Every program compiles with `loom compile`, and the emitted Rust compiles with `rustc`.

Run all proofs:
```
./run_all.sh        # Linux / WSL
python run_all.py   # Windows / cross-platform
```

## How Each Proof Works

Each folder contains:
```
proof-name/
  proof.loom        — Loom source (the claim encoded in the language)
  proof.rs          — Emitted Rust (generated by loom compile)
  README.md         — What is being proved and why it constitutes proof
  run.sh            — How to run/verify (or CI command)
```

For EMITTED experiments, `run.sh` contains the exact command to invoke the
external verifier (Kani, Prusti, Dafny, TLC).

## Claim: Loom implements these theories at the language level

The claim is not that Loom is a proof assistant. The claim is:

> A Loom program that uses `require:`/`ensure:` **cannot** be written in a way
> that violates those contracts **without the Loom compiler reporting an error**
> or the emitted Rust failing to compile.

The proof for each theory is: (1) write a correct program — it compiles; (2) write
a program that violates the theory's guarantee — the checker catches it.
Both sides are committed to each folder's `README.md`.

For the EMITTED cases, the proof is: (1) the Loom source compiles; (2) the emitted
Rust scaffold is structurally correct; (3) the external verifier (whose correctness
is independently established) accepts it.
